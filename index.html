What is Bubble Sort?

Bubble Sort is a simple sorting algorithm that repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order. The pass through the list is repeated until the list is sorted. It gets its name because smaller elements "bubble" to the top of the list.

How Does It Work?

Imagine you have an unsorted list of numbers: [5, 1, 4, 2, 8]

1. First Pass:
  • (5, 1) → Swap → [1, 5, 4, 2, 8]
  • (5, 4) → Swap → [1, 4, 5, 2, 8]
  • (5, 2) → Swap → [1, 4, 2, 5, 8]
  • (5, 8) → No swap
2. Second Pass:
  • (1, 4) → No swap
  • (4, 2) → Swap → [1, 2, 4, 5, 8]
  • (4, 5) → No swap
3. Third Pass:
  • No swaps needed → List is sorted

The algorithm stops when a complete pass is made without any swaps.

Algorithm Steps
  1. Start with an unsorted array
  2. Set a flag to track if any swaps occur
  3. For each pair of adjacent elements:
    • Compare the two elements
    • If they are in the wrong order, swap them
    • Set the swap flag to true
  4. Repeat the process until a complete pass is made without any swaps
  5. The array is now sorted
Things to Observe
•
Bubbling Up: In each pass, the largest unsorted element "bubbles up" to its correct position. The sorted portion grows from right to left.
•
Early Termination: If a full pass completes with no swaps, the algorithm stops early. Try a nearly-sorted list to see this!

What is Insertion Sort?

Insertion Sort builds the sorted array one element at a time. It picks each element from the unsorted part and inserts it into its correct position in the sorted part. It works similarly to how you sort playing cards in your hands — you pick one card and insert it in the right spot.

How Does It Work?

Imagine you have an unsorted list: [5, 3, 1, 4, 2]

1. Start: Sorted = [5], Unsorted = [3, 1, 4, 2]
  • Pick 3 → Compare with 5 → Insert before 5 → [3, 5, 1, 4, 2]
2. Sorted = [3, 5], Unsorted = [1, 4, 2]
  • Pick 1 → Shift 5, shift 3 → Insert at start → [1, 3, 5, 4, 2]
3. Sorted = [1, 3, 5], Unsorted = [4, 2]
  • Pick 4 → Shift 5 → Insert after 3 → [1, 3, 4, 5, 2]
4. Sorted = [1, 3, 4, 5], Unsorted = [2]
  • Pick 2 → Shift 5, 4, 3 → Insert after 1 → [1, 2, 3, 4, 5] ✓
Algorithm Steps
  1. Start from the second element (index 1)
  2. Store the current element as key
  3. Compare key with elements to its left:
    • Shift elements greater than key one position to the right
  4. Insert key at its correct position
  5. Repeat for all remaining elements
Things to Observe
•
Building Left: The left portion is always sorted. Watch how each new element finds its correct position.
•
Shifting: Elements shift right to make room — no full swaps, just shifts. More efficient than bubble sort in practice.

What is Selection Sort?

Selection Sort divides the array into two parts: sorted and unsorted. It repeatedly selects the minimum element from the unsorted portion and places it at the end of the sorted portion. It makes at most n−1 swaps total, which is useful when write operations are costly.

How Does It Work?

Imagine you have an unsorted list: [64, 25, 12, 22, 11]

1. First Pass — Find min in full array:
  • Min = 11 → Swap with 64 → [11, 25, 12, 22, 64]
2. Second Pass — Find min in [25, 12, 22, 64]:
  • Min = 12 → Swap with 25 → [11, 12, 25, 22, 64]
3. Third Pass — Find min in [25, 22, 64]:
  • Min = 22 → Swap with 25 → [11, 12, 22, 25, 64]
4. Fourth Pass — Find min in [25, 64]:
  • Min = 25 → Already in place → [11, 12, 22, 25, 64] ✓
Algorithm Steps
  1. Set the first element as the current minimum
  2. Scan through the unsorted portion to find the actual minimum
  3. Swap the minimum with the first unsorted element
  4. Move the boundary of the sorted portion one step right
  5. Repeat until the entire array is sorted
Things to Observe
•
Min Finding: Each pass scans the entire unsorted portion — even if it's already in order, it still scans.
•
Few Swaps: Only one swap per pass — maximum n−1 swaps total, regardless of input order.

What is Merge Sort?

Merge Sort is a divide-and-conquer algorithm. It divides the array into two halves, recursively sorts each half, and then merges the two sorted halves back into one sorted array. It guarantees O(n log n) time in all cases, making it very reliable for large datasets.

How Does It Work?

Imagine you have an unsorted list: [38, 27, 43, 3]

1. Divide Phase:
  • [38, 27, 43, 3] → [38, 27] and [43, 3]
  • [38, 27] → [38] and [27]
  • [43, 3] → [43] and [3]
2. Merge Phase (sort while merging):
  • Merge [38] + [27] → [27, 38]
  • Merge [43] + [3] → [3, 43]
  • Merge [27, 38] + [3, 43] → [3, 27, 38, 43] ✓
Algorithm Steps
  1. If the array has 1 or 0 elements, it is already sorted — return
  2. Find the midpoint and split into left and right halves
  3. Recursively apply Merge Sort to the left half
  4. Recursively apply Merge Sort to the right half
  5. Merge the two sorted halves:
    • Compare elements from both halves one by one
    • Pick the smaller one and place it in result
    • Append any remaining elements from either half
Things to Observe
•
Divide: Array splits into halves repeatedly until single elements remain — single elements are always "sorted".
•
Merge: The real sorting work happens during the merge step — two sorted arrays combine into one.
•
Extra Space: Unlike in-place sorts, Merge Sort needs O(n) extra memory to hold temporary arrays while merging.

Quick Sort
Worst O(n²) Best O(n log n) Space O(log n) Unstable ✗
What is Quick Sort?

Quick Sort is a highly efficient divide-and-conquer sorting algorithm. It selects a pivot element and partitions the array around it — elements smaller than the pivot go left, larger go right. It then recursively sorts both partitions. It is widely used in practice due to its speed and in-place nature.

How Does It Work?

Imagine you have an unsorted list: [10, 80, 30, 90, 40, 50, 70] — Pivot = 70 (last element)

1. Partition around pivot 70:
  • 10 < 70 → stays left
  • 80 > 70 → goes right
  • 30 < 70 → stays left
  • 90 > 70 → goes right
  • 40 < 70 → stays left
  • 50 < 70 → stays left
  • Place pivot 70 → [10, 30, 40, 50, 70, 80, 90]
2. Recurse on sub-arrays:
  • Sort left [10, 30, 40, 50] → already sorted
  • Sort right [80, 90] → already sorted
  • Final: [10, 30, 40, 50, 70, 80, 90] ✓
Algorithm Steps
  1. Choose a pivot element (commonly the last element)
  2. Partition the array:
    • Elements smaller than pivot → move to left side
    • Elements greater than pivot → move to right side
    • Place pivot in its correct final sorted position
  3. Recursively apply Quick Sort on the left partition
  4. Recursively apply Quick Sort on the right partition
  5. Base case: a partition of size 0 or 1 is already sorted
Things to Observe
•
Pivot Placement: After partitioning, the pivot is always in its final sorted position — it never moves again.
•
Worst Case: If the pivot is always the smallest or largest element (e.g. sorted array), performance degrades to O(n²).
•
In-place: Quick Sort sorts within the array itself — it only uses O(log n) stack space for recursion.
